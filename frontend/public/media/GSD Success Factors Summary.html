Clustering

Definition:
Clustering is an unsupervised machine learning technique used to group a set of data points into clusters such that points within the same cluster are more similar to each other than to points in other clusters. It is primarily used for exploratory data analysis.

Applications:

  • Customer segmentation in marketing.
  • Document clustering in text mining.
  • Anomaly detection in fraud detection.
  • Organizing biological data, such as gene expressions.

Types of Clustering

  1. Partitioning Clustering:
    • Divides the data into k non-overlapping subsets (clusters) where each data point belongs to exactly one cluster.
    • Algorithm: K-Means, K-Medoids.
    • Example:
      Grouping customers based on purchasing patterns, where clusters represent different customer segments.
  1. Hierarchical Clustering:
    • Builds a tree-like structure (dendrogram) of nested clusters.
    • Types:
      • Agglomerative (Bottom-Up): Starts with each data point as a separate cluster and merges them iteratively.
      • Divisive (Top-Down): Starts with one large cluster and splits it iteratively.
    • Example:
      Creating a taxonomy of animals where closely related species form smaller sub-clusters.
  1. Density-Based Clustering:
    • Forms clusters based on areas of high data point density. Points in low-density regions are considered noise.
    • Algorithm: DBSCAN (Density-Based Spatial Clustering of Applications with Noise).
    • Example:
      Identifying spatial clusters of earthquakes based on their geographic locations.
  1. Model-Based Clustering:
    • Assumes that data is generated by a mixture of probabilistic models (e.g., Gaussian distributions).
    • Algorithm: Gaussian Mixture Models (GMM).
    • Example:
      Classifying emails based on the likelihood of belonging to spam or not spam categories.
  1. Fuzzy Clustering:
    • Allows data points to belong to multiple clusters with varying degrees of membership.
    • Algorithm: Fuzzy C-Means.
    • Example:
      Grouping users of a streaming platform, where one user might belong partially to both "comedy lovers" and "action lovers" clusters.
  1. Grid-Based Clustering:
    • Divides the data space into a finite number of cells (grid structure) and performs clustering within these cells.
    • Algorithm: CLIQUE, STING.
    • Example:
      Analyzing geographical data for regional patterns, such as dividing a city map into grids and identifying clusters of urban activities.

K-Means Clustering

Overview:

  • K-Means is a partitioning clustering algorithm that divides data into kkk clusters, where kkk is pre-specified.
  • The algorithm minimizes the variance within clusters by iteratively updating cluster centroids.

Steps in K-Means:

  1. Initialization: Select kkk initial cluster centroids (randomly or based on heuristics).
  2. Assignment: Assign each data point to the nearest cluster centroid based on a distance metric (e.g., Euclidean distance).
  3. Update Centroids: Recalculate the centroid of each cluster as the mean of all points in that cluster.
  4. Repeat: Repeat steps 2 and 3 until centroids stabilize or a stopping criterion is met.

Example of K-Means:

  • Problem: Group students based on their scores in two subjects (Math and Science).
  • Steps:
    • Assume k=2k = 2k=2 clusters.
    • Assign students to clusters based on proximity to centroids.
    • Update centroids iteratively.
  • Result: Students with similar scores in Math and Science are grouped together.

Advantages:

  • Simple and efficient for large datasets.
  • Works well when clusters are spherical and well-separated.

Limitations:

  • Requires pre-specifying kkk.
  • Sensitive to outliers and initial centroid selection.

Hierarchical Clustering

Overview:

  • Hierarchical clustering builds a tree-like structure (dendrogram) representing nested clusters.
  • Two main approaches:
    1. Agglomerative (Bottom-Up): Each point starts as its own cluster, and clusters are merged iteratively.
    2. Divisive (Top-Down): Starts with one cluster containing all points, which is split recursively.

Steps in Hierarchical Clustering (Agglomerative):

  1. Initialization: Treat each data point as a single cluster.
  2. Compute Distance: Calculate pairwise distances between all clusters using a distance metric (e.g., Euclidean distance).
  3. Merge Clusters: Combine the two closest clusters based on a linkage criterion:
    • Single Linkage: Minimum distance between points in two clusters.
    • Complete Linkage: Maximum distance between points in two clusters.
    • Average Linkage: Average distance between all pairs of points in two clusters.
  4. Repeat: Continue merging clusters until only one cluster remains or a stopping criterion is met.
  5. Dendrogram: Visualize the hierarchy of clusters in a tree structure.

Example of Hierarchical Clustering:

  • Problem: Classify animals based on features like size, habitat, and diet.
  • Steps:
    • Start with individual animals as separate clusters.
    • Merge clusters iteratively based on similarity in features.
  • Result: A dendrogram shows related species grouped together.

Advantages:

  • No need to pre-specify the number of clusters.
  • Captures the hierarchy of data relationships.

Limitations:

  • Computationally expensive for large datasets.
  • Sensitive to the choice of distance metric and linkage criteria.

Hierarchical Clustering

Definition:

Hierarchical clustering is a method of grouping data points into a hierarchy of clusters. It builds a tree-like structure (dendrogram) that shows how clusters are merged or divided over successive iterations.

Types of Hierarchical Clustering

  1. Agglomerative Hierarchical Clustering (AHC):

    • Bottom-Up Approach:
      • Starts with each data point as its own cluster.
      • Gradually merges clusters until all points belong to one cluster or a stopping condition is met.
    • Example:
      Grouping books based on topics. Each book is a cluster initially; similar topics are merged into broader categories like fiction, non-fiction, etc.

  2. Divisive Hierarchical Clustering (DHC):

    • Top-Down Approach:
      • Starts with all data points in a single cluster.
      • Splits the cluster into smaller ones recursively until each point is in its own cluster.
    • Example:
      Organizing employees into departments: start with the entire company and split into smaller functional groups like HR, Finance, IT, etc.
